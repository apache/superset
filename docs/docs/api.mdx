## REST API Reference

Superset exposes a comprehensive **REST API** that follows the [OpenAPI specification](https://swagger.io/specification/).
You can use this API to programmatically interact with Superset for automation, integrations, and custom applications.

### Authentication

Most API endpoints require authentication. Superset supports several authentication methods:

1. **Session-based auth** - Log in via the UI and use the session cookie
2. **API Token** - Generate an API token from Settings > Security
3. **OAuth2** - Configure OAuth2 for programmatic access

```bash
# Example: Using curl with API token
curl -H "Authorization: Bearer YOUR_API_TOKEN" \
  https://your-superset-instance/api/v1/dashboard/
```
