## REST API Reference

Superset exposes a comprehensive **REST API** that follows the [OpenAPI specification](https://swagger.io/specification/).
You can use this API to programmatically interact with Superset for automation, integrations, and custom applications.

### Authentication

Most API endpoints require authentication. Superset supports:

1. **JWT Tokens** - Obtain via `POST /api/v1/security/login`
2. **Session-based auth** - Log in via the UI and use the session cookie
3. **API Token** - Generate from Settings &gt; Security (if enabled)

```bash
# Get a JWT token
curl -X POST http://localhost:8088/api/v1/security/login \
  -H "Content-Type: application/json" \
  -d '{"username": "admin", "password": "admin", "provider": "db"}'

# Use the token
curl -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  http://localhost:8088/api/v1/dashboard/
```
